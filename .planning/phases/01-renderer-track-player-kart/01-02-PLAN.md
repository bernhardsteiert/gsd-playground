    Track control points: 14 Vector3 control points defining an oval-with-chicane layout (use the exact coordinates from RESEARCH.md section 4 as the starting layout). Construct `const trackCurve = new THREE.CatmullRomCurve3(TRACK_POINTS, true)` — second argument `true` closes the loop.

    Road mesh: `TubeGeometry(trackCurve, 200, 5, 3, true)` — 200 tube segments, radius 5 (= half of TRACK_WIDTH 10), 3 radial segments (flat road), closed. Material: `MeshLambertMaterial({ color: 0x555555, flatShading: true })`. Add a flat ground plane: `PlaneGeometry(200, 200)` rotated -90 degrees on X, material `MeshLambertMaterial({ color: 0x228B22, flatShading: true })` (green grass). Ground plane at y = -0.1.

    Boundary walls: Loop `N_WALL_SEGS = 60` iterations. For each `i`, compute `t = i / N_WALL_SEGS`, get `point = trackCurve.getPointAt(t)` and `tangent = trackCurve.getTangentAt(t).normalize()`. Compute lateral normal as `new THREE.Vector3(-tangent.z, 0, tangent.x)`. Segment length approximation: `arcLength = trackCurve.getLength() / N_WALL_SEGS`. For each segment create a `BoxGeometry(arcLength * 1.1, 2, 0.5)` (slightly overlapping), `MeshLambertMaterial({ visible: false })` — invisible. Orient the wall mesh to align with the tangent using `mesh.lookAt(point.clone().add(tangent))`. Place outer wall at `point + normal * (5 + 0.25)` and inner wall at `point - normal * (5 + 0.25)`. Push both to `wallObjects: THREE.Mesh[]` array.

    Export: `export class Track`. Constructor takes `scene: THREE.Scene`, adds all meshes. Properties: `readonly wallObjects: THREE.Mesh[]`, `readonly waypoints: THREE.Vector3[]` (= `trackCurve.getSpacedPoints(40)`), `readonly curveRef: THREE.CatmullRomCurve3` (= `trackCurve`). Export `waypoints` and `wallObjects` as named re-exports for Phase 2 convenience: declare them at module level after construction so Phase 2 can import directly.

    The ground plane and road mesh each count as 1 draw call. 60 wall segments * 2 sides = 120 draw calls — too many with `visible:false` (invisible meshes still consume draw slots). Instead merge wall segments: use a single `InstancedMesh` for each edge (outer walls as one InstancedMesh(60 instances), inner walls as one InstancedMesh(60 instances)). `MeshLambertMaterial({ visible: false })` on InstancedMesh. Add both InstancedMesh objects to `wallObjects` but also push all 120 individual wall references into a flat `wallObjects` array for raycasting (InstancedMesh raycasting requires a different approach).

    Raycasting note: `THREE.Raycaster.intersectObject` does NOT work on InstancedMesh by default without iterating instances. Simplest solution for Phase 1: keep the 120 wall meshes as regular (non-instanced) `Mesh` with `visible: false`. Draw calls for invisible meshes are typically culled by the renderer when `visible = false` — Three.js skips rendering invisible objects, so they do NOT consume draw calls. Confirm with `renderer.info.render.calls`. This keeps the implementation simple and within the < 50 visible draw call budget.

    Export `class Kart`. Constructor takes `scene: THREE.Scene`, start position `Vector3`. Creates kart mesh group via private `createMesh()`: box body `BoxGeometry(1.2, 0.5, 2.0)` with `MeshLambertMaterial({ color: 0xff3300, flatShading: true })` positioned at `y = 0.4`; four cylinder wheels `CylinderGeometry(0.35, 0.35, 0.25, 8)` with `MeshLambertMaterial({ color: 0x222222, flatShading: true })`, each `rotation.z = Math.PI / 2`, at positions `(±0.7, 0, ±0.7)`. Group all into a `THREE.Group`. Add group to scene.

    Public `readonly state: KartState` initialized with position from constructor arg, zero velocity, heading 0, speed 0.

    Constants on the class or at module level: `MAX_SPEED = 20`, `ACCELERATION = 8`, `DRAG = 5`, `STEER_SPEED = 2.2`, `KART_HALF_WIDTH = 1.0`.

    Public `update(dt: number): void` method (reads `inputState` directly — no parameter needed):
    1. Auto-accelerate: `state.speed = Math.min(state.speed + ACCELERATION * dt, MAX_SPEED)`
    2. Apply drag: `state.speed *= (1 - DRAG * dt)`
    3. Steer: if `inputState.left` → `state.heading += STEER_SPEED * dt`; if `inputState.right` → `state.heading -= STEER_SPEED * dt`
    4. Compute velocity: `state.velocity.set(Math.sin(state.heading) * state.speed, 0, Math.cos(state.heading) * state.speed)`
    5. Tentative next pos: `nextPos = state.position.clone().addScaledVector(state.velocity, dt)`
    6. Wall raycast: call private `checkWall(nextPos)`. If hit: compute `dot = state.velocity.dot(normal)`, if `dot < 0` → `state.velocity.addScaledVector(normal, -dot * 1.2)`. Recompute `nextPos = state.position.clone().addScaledVector(state.velocity, dt)`.
    7. `state.position.copy(nextPos)`
    8. Sync mesh: `this.mesh.position.copy(state.position)`, `this.mesh.rotation.y = state.heading`

    Private `checkWall(pos: THREE.Vector3): { hit: boolean; normal: THREE.Vector3 | null }`:
    - Create `_raycaster = new THREE.Raycaster()` (reuse per-instance, not per-call).
    - Left direction: `new THREE.Vector3(Math.cos(state.heading), 0, -Math.sin(state.heading))`
    - Right direction: left direction negated
    - For each direction: `_raycaster.set(pos, dir)`, intersect with `wallObjects`. If `hits[0].distance < KART_HALF_WIDTH` → return `{ hit: true, normal: hits[0].face!.normal.clone() }`.
    - Return `{ hit: false, normal: null }` if no hit.

    Note: `wallObjects` is imported as the module-level export from `Track.ts`. This creates an import dependency. Because Track.ts is instantiated in main.ts before Kart.ts, the array is populated at module execution time — this is fine as long as Kart is constructed after Track.

    In constructor: create `this.anchor = new THREE.Object3D()`. Call `scene.add(this.anchor)`. Do NOT add camera to the anchor as a child — the camera passed in from main.ts is already in the scene graph (added by Three.js internals or main.ts). Instead, manage the camera position directly each frame by computing world-space target and setting camera.position + camera.lookAt.

    Rationale for not child-parenting: parenting requires careful coordinate space handling. The simpler approach that matches D-09's anti-jitter goal is to lerp `this.anchor.position` toward the target position, then set `camera.position.copy(this.anchor.position)` + call `camera.lookAt(lookTarget)` each frame. This achieves the same jitter-absorbing effect without the coordinate space complexity.

    Constant: `LERP_FACTOR = 0.08`.

    Public `update(dt: number, state: KartState): void`:

    3. `this.anchor.position.lerp(targetPos, LERP_FACTOR)`
    4. `camera.position.copy(this.anchor.position)`
    5. Look-ahead target: `state.position.clone().addScaledVector(state.velocity, 0.5)` — aims where the kart will be in 0.5s
    6. `camera.lookAt(lookTarget)`
